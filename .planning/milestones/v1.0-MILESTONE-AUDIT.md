### Integration Notes (non-blocking)

**1. No HTTP server in stdout mode (OBS-01, OBS-02)**

`runPipeline` starts no HTTP server in `--output stdout` mode. `/metrics` and `/healthz` are unreachable in the default mode. Metrics counters are incremented correctly in all modes — but cannot be scraped in stdout. Recommended v1.1 fix: start a background HTTP server on a configurable metrics port regardless of output mode.

**2. connector.Events() orphaned channel (informational)**

`PostgresConnector.Events()` is exported and written to non-blocking but never consumed by `runPipeline`. Events flow through EventLog → Router instead. Intentional per connector.go:218 comment. No functional impact.

**3. CFG-02 single-connector at runtime (informational)**

YAML config supports multi-table per-source settings fully. However, one `runPipeline` invocation = one `PostgresConnector`. True simultaneous multi-source requires multiple processes. Config-layer requirement is satisfied.
