## Cross-Phase Integration

### All Flows Verified (4/4)

**Flow 1: Postgres CDC, HA mode (HA-01, HA-02, HA-03, CHK-05)**
INT-03 guard passes for Postgres DSN → `ha.NewLeaderElector` → `RunStandby` blocks until lock → `OpenPostgres` loads shared LSN → Postgres CDC pipeline. No regression from Phase 9.1. ✓

**Flow 2: MongoDB CDC, non-HA (SRC-09–12, PAR-04)**
MongoDB dispatch → `runMongoPipeline` → `NormalizeChangeEvent` → `AppendAndQueue` (CHK-01) → watermark-gated snapshot on token expiry. ✓

**Flow 3: MongoDB re-snapshot after InvalidResumeToken (SRC-11)**
`isInvalidResumeToken` → `NeedsSnapshot()=true` → `MongoSnapshot.Run` → connector restart. ✓

**Flow 4: MongoDB + HA rejected (INT-03)**
`--ha` + `mongodb://` → guard at root.go:222–224 fires before any pgx call → clear error `"ha: --ha requires a Postgres source DSN; MongoDB source detected (%s)"`. `TestMongoDBWithHAReturnsError` passes. ✓

### Advisory Finding (non-blocking)

**Health probe "postgres" fires for MongoDB sources (root.go:341–353)**
The health probe added in Phase 8-03 calls `pgx.Connect(ctx, cfg.Source)` unconditionally. For MongoDB deployments, `/healthz` permanently reports the `postgres` probe unhealthy. Pipeline operation is unaffected. Pre-existing, not introduced by Phase 9.1.
Recommendation: guard the probe — `if cfg.SourceType() == "postgres" { healthProbes = append(...) }`.
