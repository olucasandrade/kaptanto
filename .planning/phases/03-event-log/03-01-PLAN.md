    Implement key encoding helpers. ALL numeric key components must be fixed-width big-endian binary — NEVER decimal ASCII (see research pitfall 1):
    - encodePartKey(partition uint32, seq uint64) []byte → [0x50][4 bytes partition BE][0x53][8 bytes seq BE] (14 bytes total)
    - encodePartPrefix(partition uint32) []byte → [0x50][4 bytes partition BE] (5 bytes — for ValidForPrefix)
    - encodeDedupKey(idempotencyKey string) []byte → [0x44][idempotencyKey bytes]
    - encodePartSeq(partition uint32, seq uint64) []byte → 12 bytes (4+8 BE) — stored as dedup value
    - decodePartKey(key []byte) (partition uint32, seq uint64) → extract from 14-byte key

    **Step 4: internal/eventlog/badger.go**
    Implement BadgerEventLog struct:
    ```
    type BadgerEventLog struct {
        db            *badger.DB
        seqs          []*badger.Sequence
        numPartitions uint32
        retention     time.Duration
    }
    ```

    Open(dir string, numPartitions uint32, retention time.Duration) (*BadgerEventLog, error):
    - badger.DefaultOptions(dir).WithLogger(nil) — suppress Badger logger noise (research pitfall 5)
    - badger.Open(opts)
    - For each partition i: db.GetSequence(fmt.Appendf(nil, "seq:p:%d", i), 256)
    - Return BadgerEventLog

    Append(ev *event.ChangeEvent) (uint64, error):
    - partitionOf(ev.Key, b.numPartitions) using FNV-1a: hash/fnv New32a, write ev.Key bytes, Sum32() % numPartitions
    - json.Marshal(ev) for value
    - dedupKey = encodeDedupKey(ev.IdempotencyKey)
    - IMPORTANT: call b.seqs[partition].Next() OUTSIDE the transaction (see research anti-patterns — seq.Next() before db.Update)
    - db.Update(func(txn) error):
        - txn.Get(dedupKey): if err == nil → already exists, return nil (seq sentinel = 0 already gotten, just don't write)
        - if err != badger.ErrKeyNotFound → return wrapped error
        - partKey = encodePartKey(partition, seq)
        - txn.SetEntry(badger.NewEntry(partKey, val).WithTTL(b.retention))
        - txn.SetEntry(badger.NewEntry(dedupKey, encodePartSeq(partition, seq)).WithTTL(b.retention))
        - Both entries get SAME TTL (research pitfall 4)
    - Return seq, err

    NOTE on dedup sentinel: when dedup detects duplicate, seq was already consumed from the sequence. Return seq=0 as "already existed" sentinel (callers that need to distinguish can check seq==0). This is documented behavior — sequences need not be gapless.

    ReadPartition(ctx context.Context, partition uint32, fromSeq uint64, limit int) ([]LogEntry, error):
    - db.View(func(txn) error):
        - opts = badger.DefaultIteratorOptions; opts.Prefix = encodePartPrefix(partition)
        - it.Seek(encodePartKey(partition, fromSeq))
        - Loop: it.ValidForPrefix(prefix) && len(entries) < limit:
            - ctx.Err() check
            - item.ValueCopy(nil) → json.Unmarshal into &event.ChangeEvent
            - _, seq := decodePartKey(item.KeyCopy(nil))
            - append LogEntry{Seq: seq, Event: &ev}
    - Return entries, err

    Close() error:
    - For each seq: seq.Release() (research pitfall 6)
    - return b.db.Close()

    partitionOf helper (unexported):
    - h := fnv.New32a(); h.Write(groupingKey); return h.Sum32() % numPartitions
