  <behavior>
    All behavior is testable without a live Postgres connection. Use mock EventLog and BackfillStore.

    **BKF-01 — Keyset cursor queries (cursor.go):**
    - buildFirstQuery("orders", ["id"], 5000) → `SELECT * FROM orders ORDER BY id ASC LIMIT 5000`
    - buildNextQuery("orders", ["id"], []any{99}, 5000) → `SELECT * FROM orders WHERE id > $1 ORDER BY id ASC LIMIT 5000`, args=[99]
    - buildNextQuery("orders", ["tenant_id","id"], []any{"acme",99}, 500) → `SELECT * FROM orders WHERE (tenant_id, id) > ($1, $2) ORDER BY tenant_id ASC, id ASC LIMIT 500`, args=["acme",99]
    - NEVER emit a query containing "OFFSET"

    **BKF-02 — Watermark dedup (watermark.go):**
    - WatermarkChecker.ShouldEmit: for table "orders", pk=`{"id":1}`, snapshotLSN=100:
      - Returns true when EventLog.ReadPartition returns no entries for that partition
      - Returns true when all partition entries have metadata["lsn"] <= 100
      - Returns false when any partition entry has Table=="orders", Key==`{"id":1}`, metadata["lsn"]==200 (> snapshotLSN)
    - Must call PartitionOf(pk, numPartitions) to select correct partition (not scan all partitions)

    **BKF-03 — Crash-resumable state (store.go):**
    - SQLiteBackfillStore.SaveState persists BackfillState to backfill_states table (upsert by source_id+table_name)
    - SQLiteBackfillStore.LoadState returns nil, nil when no row exists (first run)
    - After SaveState(state{CursorKey: `{"id":99}`, ProcessedRows: 5000, Status: "running"}), a fresh LoadState returns the same values

    **BKF-04 — Adaptive batch sizing (optimizer.go):**
    - BatchOptimizer.Adjust(500ms) → returns current * 1.25, capped at 50000
    - BatchOptimizer.Adjust(4s) → returns current / 2, floored at 100
    - BatchOptimizer.Adjust(6s) → returns current / 2, floored at 100
    - BatchOptimizer starts at current=5000 (defaultBatch)
    - Adjust(2s) → returns 5000 unchanged (between 1s and 3s thresholds: no change)

    **BKF-05 — All five strategies (backfill.go):**
    - BackfillConfig.Strategy == "stream_only": BackfillEngine.HasPendingBackfills() returns false; LoadState returns state with Status=="completed"
    - BackfillConfig.Strategy == "snapshot_deferred": state persisted as Status=="deferred"; Run() returns immediately without executing snapshot loop
    - Other strategies ("snapshot_and_stream", "snapshot_only", "snapshot_partial"): HasPendingBackfills() returns true when state is pending/running

    **EVT-03 — Snapshot read event shape (backfill.go):**
    - makeReadEvent(sourceID="pg1", schema="public", table="orders", pkJSON=`{"id":1}`, rowJSON=`{"id":1,"name":"foo"}`, state) returns ChangeEvent with:
      - Operation == event.OpRead
      - Before == nil (json.RawMessage(nil))
      - After == rowJSON
      - Key == pkJSON
      - IdempotencyKey format: "pg1:public.orders:{\"id\":1}:read:snap_{snapshotID}"
      - Metadata["snapshot"] == true
      - Metadata["snapshot_id"] != ""
      - Metadata["snapshot_progress"]["total"] == state.TotalRows
      - Metadata["snapshot_progress"]["completed"] == state.ProcessedRows

    **EVT-04 — Control event shape (backfill.go):**
    - makeControlEvent(sourceID, table, "snapshot_complete", state) returns ChangeEvent with:
      - Operation == event.OpControl
      - Before == nil, After == nil
      - Key == json.RawMessage(`{}`)
      - Metadata["control_type"] == "snapshot_complete"
      - Metadata["total_rows"] == state.ProcessedRows
      - Metadata["snapshot_id"] != ""

    **Step 2: internal/backfill/optimizer.go — BatchOptimizer**

    ```go
    const (
        defaultBatch = 5_000
        minBatch     = 100
        maxBatch     = 50_000
    )

    type BatchOptimizer struct{ current int }

    func NewBatchOptimizer() *BatchOptimizer { return &BatchOptimizer{current: defaultBatch} }

    func (o *BatchOptimizer) Adjust(d time.Duration) int {
        switch {
        case d < time.Second:
            o.current = min(int(float64(o.current)*1.25), maxBatch)
        case d > 5*time.Second:
            o.current = max(o.current/2, minBatch)
        case d > 3*time.Second:
            o.current = max(o.current/2, minBatch)
        }
        return o.current
    }

    func (o *BatchOptimizer) Current() int { return o.current }
    ```
