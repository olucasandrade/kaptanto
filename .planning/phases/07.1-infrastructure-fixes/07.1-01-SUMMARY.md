# Phase 07.1 Plan 01: CHK-02 LogEntry.PartitionID Propagation Summary

**One-liner:** Added `PartitionID uint32` to `LogEntry`, populated by `ReadPartition`, used by `SSEConsumer.Deliver` at all three `SaveCursor` callsites — fixing the multi-partition cursor re-delivery bug.

## What Was Built

Fixed CHK-02: `SSEConsumer.Deliver` was passing `partitionID=0` hardcoded to `SaveCursor` because `LogEntry` lacked a `PartitionID` field. Consumers on partitions > 0 would re-deliver all events from those partitions on reconnect.

**Changes:**
- `internal/eventlog/eventlog.go`: Added `PartitionID uint32` field to `LogEntry` struct (after `Seq`, before `Event`, with doc comment)
- `internal/eventlog/badger.go`: Updated `ReadPartition` to set `PartitionID: partition` in every `LogEntry` appended
- `internal/output/sse/consumer.go`: Replaced all three `SaveCursor(..., 0, ...)` calls with `entry.PartitionID`; removed all three TODO comments
- `internal/output/sse/consumer_test.go`: Added `TestSSEConsumer_Deliver_PartitionID` with 3 sub-tests (success path, EventFilter rejection, RowFilter rejection), each asserting `partitionID=3` is passed through and `partitionID=0` receives nothing
