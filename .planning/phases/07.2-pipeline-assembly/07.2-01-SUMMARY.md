# Phase 07.2 Plan 01: Server-Layer Per-Table Filter Maps Summary

**One-liner:** SSEServer and GRPCServer upgraded to accept and thread per-table map[string]*RowFilter and map[string][]string through consumer constructors, fixing the nil, nil passthrough gap for CFG-05 and CFG-06.

## Tasks Completed

| Task | Name | Commit | Files |
|------|------|--------|-------|
| 1 | Add per-table filter maps to SSEServer and update ServeHTTP | 8275767 | sse/consumer.go, sse/server.go, sse/consumer_test.go |
| 2 | Add per-table filter maps to GRPCServer and update Subscribe | a8353df | grpc/consumer.go, grpc/server.go, grpc/consumer_test.go |

## What Changed

### SSE (Task 1)

- `SSEConsumer` struct: replaced `rowFilter *output.RowFilter` and `allowedColumns []string` with `rowFilters map[string]*output.RowFilter` and `colFilters map[string][]string`
- `NewSSEConsumer`: last two params changed from single values to maps
- `SSEConsumer.Deliver`: row filter uses `c.rowFilters[entry.Event.Table]` lookup; column filter uses `c.colFilters[entry.Event.Table]`
- `SSEServer` struct: added `rowFilters` and `colFilters` fields
- `NewSSEServer`: accepts two new params `rowFilters` and `colFilters` after `pingInterval`
- `ServeHTTP`: passes `s.rowFilters` and `s.colFilters` to `NewSSEConsumer` (was `nil, nil`)
- `consumer_test.go`: 4 call sites with specific filters updated to use table-keyed maps (`"orders"` key)

### gRPC (Task 2)

- `GRPCConsumer` struct: replaced `rowFilter *output.RowFilter` and `allowedColumns []string` with `rowFilters map[string]*output.RowFilter` and `colFilters map[string][]string`
- `NewGRPCConsumer`: last two params changed from single values to maps
- `GRPCConsumer.Deliver`: row filter uses `c.rowFilters[entry.Event.Table]` lookup; column filter uses `c.colFilters[entry.Event.Table]`
- `GRPCServer` struct: added `rowFilters` and `colFilters` fields
- `NewGRPCServer`: accepts two new params `rowFilters` and `colFilters` after `metrics`
- `Subscribe`: passes `s.rowFilters` and `s.colFilters` to `NewGRPCConsumer` (was `nil, nil`)
- `consumer_test.go`: 4 call sites with specific filters updated to use table-keyed maps (`"orders"` key)

## Decisions Made

1. **Per-table maps on consumer structs** — Deliver has access to `entry.Event.Table`; looking up filter by table at delivery time is zero-overhead and correct for multi-table SSE/gRPC streams.

2. **nil map is pass-through** — Go's nil map lookup returns the zero value (`nil, false`) without panicking. No nil guards needed for the maps themselves.

3. **NewSSEServer/NewGRPCServer no external callers yet** — Plan 02 will call `buildTableFilters` and pass the resulting maps. The signature change is backward-compatible at the binary level.
