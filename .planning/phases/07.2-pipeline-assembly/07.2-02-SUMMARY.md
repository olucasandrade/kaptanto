## Decisions Made

- Used `defer el.Close()` at the top of runPipeline (after Open) rather than inside g.Wait — Go defer semantics guarantee it runs after g.Wait() returns, satisfying the "Badger closes after Router stops" invariant
- `golang.org/x/sync` was already in go.sum as a transitive dep but not importable without a direct `require` line in go.mod; ran `go get golang.org/x/sync@v0.19.0` to promote it
- `cursorStore.Run(gctx)` wrapped with `g.Go(func() error { ...; return nil })` — Run has no error return, always exits on context cancellation
- gRPC `lis` declared with `var lis net.Listener` then assigned with `=` to avoid shadowing the outer `err` variable already in scope from sql.Open
