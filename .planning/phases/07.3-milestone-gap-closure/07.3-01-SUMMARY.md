**Drain-or-drop select replaces blocking channel send in AppendAndQueue so the WAL receive loop can never stall when connector.Events() is not drained**

## Accomplishments
- Replaced the blocking `select{case c.events<-ev; case <-ctx.Done(): return ctx.Err()}` with `select{case c.events<-ev; default:}` — eliminating the only blocking path in the WAL receive goroutine
- Added `TestAppendAndQueue_NonBlockingWhenChannelFull` which sends 2000 events without draining the channel (buffer=1024) and asserts: (a) no deadlock, (b) all 2000 reach eventLog.Append (LOG-01), (c) returns nil for all calls
- All 12 existing connector tests continue to pass — no regression

## Decisions Made
- Drain-or-drop (`default:` arm) chosen over other approaches — event is already durable in Badger before the send, so dropping from the channel is safe; Router reads from `eventLog.ReadPartition` not `connector.Events()`
- `ctx context.Context` parameter kept in `AppendAndQueue` signature unchanged — only the select body changed, zero caller breakage
- Removed `case <-ctx.Done(): return ctx.Err()` arm — no blocking path remains, ctx cancellation check is now unnecessary in this select
