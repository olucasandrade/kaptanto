# Phase 7.6 Plan 01: Backfill Correctness Fixes Summary

Three surgical fixes for audit-identified bugs in the backfill subsystem: SnapshotLSN never assigned (BKF-02), URI pragma format in SQLiteBackfillStore (BKF-03), concurrent Run goroutines when both HasPendingBackfills and needsSnapshot are true (SRC-06).

## Changes Per File

### internal/backfill/backfill.go
- Added `"github.com/jackc/pglogrepl"` import
- In `snapshotTable`, after first `SaveState` call and before the batch loop: inserted `if state.SnapshotLSN == 0` block that queries `pg_current_wal_flush_lsn()` and parses the result via `pglogrepl.ParseLSN`; non-fatal on query/parse failure (leaves SnapshotLSN=0, conservative path); guard prevents overwrite on crash-resume

### internal/backfill/store.go
- `OpenSQLiteBackfillStore`: replaced URI pragma DSN (`file://...?_pragma=journal_mode(WAL)&_pragma=synchronous(NORMAL)`) with `sql.Open("sqlite", path)` followed by `db.Exec` loop matching `internal/checkpoint/sqlite.go` reference pattern

### internal/backfill/backfill_test.go
- Added imports: `database/sql`, `fmt`, `_ "modernc.org/sqlite"`
- `TestSnapshotLSN_NotOverwrittenOnResume`: pre-populates state with non-zero SnapshotLSN, runs engine with failing openConnFn, asserts persisted SnapshotLSN unchanged
- `TestOpenSQLiteBackfillStore_WALModeApplied`: opens store, saves/loads state, then re-opens raw db and queries `PRAGMA journal_mode` to assert "wal"

### internal/source/postgres/connector.go
- Merged blocks 12a and 12b into single goroutine launch: `if c.backfillEng != nil && (c.backfillEng.HasPendingBackfills() || needsSnapshot)`
- Logs `needs_snapshot` field for operational triage
- Added `TODO(SRC-06)` comment for future cursor-reset consideration

### internal/source/postgres/connector_test.go
- `TestSRC06_SingleRunGoroutineWhenBothConditionsTrue`: verifies the merged boolean expression covers all four combinations; confirms connector wiring with mock
