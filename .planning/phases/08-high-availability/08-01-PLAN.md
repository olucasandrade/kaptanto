<feature>
  <name>PostgresStore — Postgres-backed CheckpointStore</name>
  <files>internal/checkpoint/postgres.go, internal/checkpoint/postgres_test.go</files>
  <behavior>
    - OpenPostgres(ctx, dsn string) creates table postgres_checkpoints if not exists, returns *PostgresStore
    - Save(ctx, sourceID, lsn) upserts row: INSERT INTO postgres_checkpoints(source_id, lsn) VALUES($1,$2) ON CONFLICT(source_id) DO UPDATE SET lsn=EXCLUDED.lsn, updated_at=NOW()
    - Load(ctx, sourceID) returns ("", nil) when no row exists (first run) — not an error
    - Load(ctx, sourceID) returns stored lsn string when row exists
    - Close() calls conn.Close(ctx) and releases the connection
    - Ping() opens a short context and pings the connection — used by health handler
    - PostgresStore satisfies CheckpointStore interface — add compile-time check: var _ CheckpointStore = (*PostgresStore)(nil)
    - Table schema: source_id TEXT PRIMARY KEY, lsn TEXT NOT NULL, updated_at TIMESTAMPTZ DEFAULT NOW()
    - Test: Save then Load returns same lsn
    - Test: Load on empty table returns ("", nil)
    - Test: Save twice updates existing row (no duplicate)
    - Test: Uses testcontainers or skips with t.Skip("no POSTGRES_TEST_DSN") env var — check os.Getenv("POSTGRES_TEST_DSN"); if empty, t.Skip("set POSTGRES_TEST_DSN to run Postgres integration tests")
  </behavior>
  <implementation>
    Use pgx.Connect(ctx, dsn) — single connection, not a pool. HA mode runs one instance per process so connection overhead is minimal. pgxpool adds complexity (idle connections) without benefit here.

    Do NOT use database/sql with pgx driver — use pgx native API (pgx.Conn directly) for consistency with the existing postgres/connector.go pattern in this codebase.

    Keep OpenPostgres signature consistent with Open() on SQLiteStore: takes a DSN string (not a *pgx.Conn) so callers in runPipeline can use cfg.Source directly.

    Ping() uses context.WithTimeout(2s) — mirrors the postgres health probe in root.go.
  </implementation>
</feature>
