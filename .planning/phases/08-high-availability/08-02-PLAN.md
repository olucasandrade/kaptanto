Available in go.mod:
- github.com/jackc/pgx/v5 v5.5.4 — pgx.Connect(ctx, dsn) returns (*pgx.Conn, error)
- pgx.Conn.Exec(ctx, sql, args...) — for advisory lock calls
- pgx.Conn.QueryRow(ctx, sql, args...).Scan(&result) — for SELECT pg_try_advisory_lock($1)

Postgres advisory lock functions:
- SELECT pg_try_advisory_lock($1) — non-blocking, returns bool true if acquired
- SELECT pg_advisory_unlock($1) — explicit release; returns bool
- Lock ID: use a fixed int64 constant, e.g. 0x4B415054414E544F (hex for "KAPTANTO")
- Lock is session-scoped: automatically released when the pgx.Conn is closed

<feature>
  <name>LeaderElector — Postgres advisory lock leader election</name>
  <files>internal/ha/leader.go, internal/ha/leader_test.go</files>
  <behavior>
    - NewLeaderElector(ctx context.Context, dsn string) (*LeaderElector, error) — opens a dedicated pgx.Conn for the HA connection (NOT the replication connection); returns error if connection fails
    - TryAcquire(ctx context.Context) (bool, error) — executes SELECT pg_try_advisory_lock($1) with lockID constant; returns (true, nil) if lock acquired, (false, nil) if another session holds it
    - Release(ctx context.Context) error — executes SELECT pg_advisory_unlock($1); returns error if unlock fails
    - RunStandby(ctx context.Context, pollInterval time.Duration) error — polls TryAcquire every pollInterval until: (a) ctx is cancelled → return ctx.Err(), or (b) TryAcquire returns true → return nil to signal "I am now the leader"
    - Close() — closes the pgx.Conn; session-scoped lock is released automatically by Postgres
    - lockID constant: const haLockID int64 = 0x4B415054414E544F (KAPTANTO in hex)
    - Test: single connection TryAcquire returns true (no contention)
    - Test: two connections — second TryAcquire returns false while first holds lock
    - Test: first connection closed → second connection TryAcquire returns true (simulates crash takeover)
    - Test: RunStandby returns nil after leader closes connection within pollInterval (use 50ms poll in tests)
    - Test: RunStandby returns ctx.Err() when ctx is cancelled before lock is acquired
    - Tests skip with t.Skip("set POSTGRES_TEST_DSN") if os.Getenv("POSTGRES_TEST_DSN") == ""
  </behavior>
  <implementation>
    Use a single dedicated pgx.Conn for the HA lock — separate from the replication connection and the Postgres checkpoint store connection. This is intentional: the lock must be held for the full lifetime of the leader process.

    RunStandby uses time.NewTicker(pollInterval) inside a select loop — ctx.Done() and ticker.C arms. On ticker fire: call TryAcquire; on true return nil; on error log and continue polling (transient DB hiccup should not stop the standby).

    Do NOT use pg_advisory_lock() (blocking) — use pg_try_advisory_lock() (non-blocking) so RunStandby can respect ctx cancellation cleanly.

    Package path: internal/ha — new package, create the directory.
  </implementation>
</feature>
