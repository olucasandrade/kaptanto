Replace the `if cfg.HA { slog.Warn(...) }` stub in runPipeline with the following HA branch:

```
if cfg.HA {
    // 1. Create LeaderElector with dedicated HA connection
    elector, err := ha.NewLeaderElector(ctx, cfg.Source)
    if err != nil {
        return fmt.Errorf("ha: connect for leader election: %w", err)
    }
    defer elector.Close()

    // 2. Compete for advisory lock — blocks in standby until acquired or ctx cancelled
    slog.Info("ha: entering standby, polling for advisory lock")
    if err := elector.RunStandby(ctx, 2*time.Second); err != nil {
        return fmt.Errorf("ha: standby interrupted: %w", err)
    }
    slog.Info("ha: advisory lock acquired — this instance is now the leader")
}
```

After the HA block, use cfg.HA to select the checkpoint store implementation:
- If cfg.HA=true: call `checkpoint.OpenPostgres(ctx, cfg.Source)` instead of `checkpoint.Open(path)`.
  - The pgStore satisfies CheckpointStore; assign to ckStore variable.
  - Defer pgStore.Close() (same pattern as SQLiteStore).
  - Add an HA health probe to healthProbes slice: `{Name: "ha_lock", Check: func() error { _, err := pgStore.Ping(); return err }}` — but since PostgresStore.Ping() already returns error directly, use `pgStore.Ping`.
- If cfg.HA=false: keep existing `checkpoint.Open(filepath.Join(cfg.DataDir, "checkpoint.db"))` (no change).

The existing healthProbes slice is built as a literal in runPipeline step 8. Refactor it to be built incrementally so the HA probe can be appended conditionally:

```go
healthProbes := []observability.HealthProbe{
    {Name: "badger", Check: el.Ping},
    {Name: "checkpoint", Check: ckProbe}, // ckProbe set per-path below
    {Name: "cursors", Check: cursorStore.Ping},
    {Name: "postgres", Check: postgresConnCheck},

if cfg.HA {
    healthProbes = append(healthProbes, observability.HealthProbe{Name: "ha_lock", Check: pgStore.Ping})

healthHandler := observability.NewHealthHandler(healthProbes)
```

Where `ckProbe` is `ckStore.Ping` for SQLite or `pgStore.Ping` (already assigned) for HA path. Since PostgresStore and SQLiteStore both implement Ping() error, this is straightforward.

Note: RunStandby with pollInterval=2*time.Second means standby checks every 2 seconds. When the leader crashes, the Postgres session closes and the advisory lock is released; the standby acquires it within 2 seconds (HA-02).

Note: After lock acquisition, runPipeline opens Postgres checkpoint store and calls ckStore.Load("default") inside the connector setup — this gives the new leader the last saved LSN (HA-03).
