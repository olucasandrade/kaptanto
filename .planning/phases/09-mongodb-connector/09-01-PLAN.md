    ```go
    // Config holds all parameters for the MongoDBConnector.
    type Config struct {
        URI         string   // MongoDB connection string, e.g. "mongodb://localhost:27017"
        Database    string   // Target database name
        Collections []string // Collections to watch (required, no wildcards)
        SourceID    string   // Stable identifier for checkpoint keying; defaults to "mongo_default"
    }

    func (c *Config) ApplyDefaults() {
        if c.SourceID == "" { c.SourceID = "mongo_default" }
    }

    type MongoDBConnector struct {
        cfg          Config
        store        checkpoint.CheckpointStore
        idGen        *event.IDGenerator
        eventLog     eventlog.EventLog        // nil if no durable log
        events       chan *event.ChangeEvent
        needsSnapshot bool
        // watchFn allows test injection: defaults to real mongo collection Watch call
        watchFn      func(ctx context.Context, coll string, resumeToken bson.Raw) (changeStreamIter, error)
    }
    ```

    Key implementation points:
    - `changeStreamIter` is a local interface: `Next(ctx) bool; Decode(v any) error; ResumeToken() bson.Raw; Err() error; Close(ctx) error`
    - On construction, call store.Load to get the last resume token. Store it as `c.resumeToken bson.Raw`.
    - `Run(ctx)`: for each collection in cfg.Collections, start a goroutine that calls c.watchFn(ctx, coll, c.resumeToken) and loops on iter.Next(ctx). On each event, call c.processEvent (parse + AppendAndQueue + save token). On error, check for InvalidResumeToken error code (260) — if yes, set c.needsSnapshot=true and return nil. Otherwise log and retry with backoff (2s initial, 60s max).
    - `processEvent` is responsible for: decoding the raw change stream event, constructing a ChangeEvent via the normalizer (call mongodb.NormalizeChangeEvent — defined in Plan 02's normalizer package, but for Plan 01 use a simple stub that you can replace in Plan 03), then calling AppendAndQueue.
    - For Plan 01, the normalizer stub just sets operation from operationType field and leaves Key/Before/After as raw JSON. The real normalizer (Plan 02) will replace this.
    - `AppendAndQueue`: if c.eventLog != nil, call el.Append(ctx, ev) first; return error if it fails (do NOT save token). Then call store.Save(ctx, c.cfg.SourceID, tokenJSON). Then send ev to c.events channel (non-blocking with drain-or-drop select, matching [Phase 07.3-milestone-gap-closure] pattern).
    - `NeedsSnapshot() bool` — simple accessor.
    - `Events() <-chan *event.ChangeEvent` — returns the channel.
    - Use `go.mongodb.org/mongo-driver/v2/mongo`, `go.mongodb.org/mongo-driver/v2/bson`, `go.mongodb.org/mongo-driver/v2/mongo/options`.
    - InvalidResumeToken detection: `mongo.IsCommandError(err, 260)` OR strings.Contains(err.Error(), "InvalidResumeToken").
