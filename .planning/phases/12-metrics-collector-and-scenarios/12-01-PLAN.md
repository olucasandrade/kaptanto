  <action>
    Create `bench/cmd/collector/main.go` in package `main`.

    CLI flags (use `flag` stdlib, matching loadgen pattern):
    - `--output` (string, default `"metrics.jsonl"`) — path for NDJSON output
    - `--kaptanto-url` (string, default `"http://localhost:7654/stream?consumer=collector"`) — Kaptanto SSE URL
    - `--debezium-port` (int, default `8081`) — HTTP port for Debezium webhook receiver
    - `--sequin-port` (int, default `8082`) — HTTP port for Sequin push receiver
    - `--kafka-brokers` (string, default `"localhost:9092"`) — comma-separated Redpanda brokers
    - `--kafka-topic` (string, default `"public.bench_events"`) — PeerDB Kafka topic
    - `--management-port` (int, default `8080`) — HTTP port for scenario management API (`/scenario`, `/scenario/last-event`)

    Startup sequence:
    1. Create `records := make(chan collector.EventRecord, 10000)` — shared buffered channel
    2. Create `var scenario atomic.Value; scenario.Store("unknown")` — adapters read this
    3. Create `lastSeen := struct{ mu sync.Mutex; m map[string]time.Time }{m: make(map[string]time.Time)}` — tracks last receive_ts per tool; updated by a fan-out goroutine reading from `records` before forwarding to writer
    4. Create `ctx, cancel := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)`
    5. Start writer goroutine: `go collector.RunWriter(ctx, *output, records)`
    6. Start Kaptanto adapter goroutine: `go adapters.Run(ctx, *kaptantoURL, &scenario, records)`
    7. Start HTTP mux for Debezium on `--debezium-port` and Sequin on `--sequin-port`. Use separate `http.ServeMux` per port.
    8. Start PeerDB adapter goroutine: parse `--kafka-brokers` by splitting on comma
    9. Start management API on `--management-port` with its own `http.ServeMux`:
       - `POST /scenario?name=X` — calls `scenario.Store(name)`, returns 200
       - `GET /scenario` — returns current scenario name as plain text, returns 200
       - `GET /scenario/last-event?tool=X` — acquires `lastSeen.mu`, returns JSON `{"tool":"X","last_receive_ts":"<RFC3339Nano>"}`. If tool not found, returns `{"tool":"X","last_receive_ts":""}` with 200 (not 404, to avoid crash-recovery poll logic treating 404 as fatal).
    10. Log "collector: ready (management=:%d, debezium=:%d, sequin=:%d)" to stderr
    11. Block on `<-ctx.Done()`; log "collector: shutting down"

    The `lastSeen` map is updated inside a fan-out goroutine that sits between adapters and the writer channel:
    ```go
    // fanout goroutine: reads from adapters' direct channel, updates lastSeen, forwards to writer channel
    go func() {
        for rec := range adapterCh {
            lastSeen.mu.Lock()
            lastSeen.m[rec.Tool] = rec.ReceiveTS
            lastSeen.mu.Unlock()
            records <- rec
        }
    }()
    ```
    Adapters send to `adapterCh` (buffered 10000); the fan-out goroutine forwards to `records` (also buffered 10000) consumed by the writer. This keeps the management API reads consistent without blocking adapter goroutines.

    Note: `statsd` binary (12-02) runs separately; collector does NOT start statsd.
