    - `Runner` struct with fields: `CollectorURL`, `LoadgenBin`, `DSN`, `OutputDir`

    Implement `Runner` methods:

    `(r *Runner) Init(ctx context.Context) error`:
    - Create PeerDB peer and mirror: `exec.Command("psql", "-h", "localhost", "-p", "9900", "-U", "peerdb", "-d", "peerdb", "-c", sql)` for each SQL statement. Log errors but continue (PeerDB may already have peer from a previous run).

    - Log "scenarios: init complete"

    `(r *Runner) Run(ctx context.Context, scenarios []ScenarioDef) error`:
    - For each scenario in sequence:
      1. `r.setScenarioTag(ctx, s.Name)` — POST to collector /scenario endpoint
      2. `r.writeMarker(s.Name, "start")` — append `{"scenario_event":"start","scenario":"X","ts":"..."}` to outputDir/metrics.jsonl
      3. If `s.PreWaitS > 0`: `time.Sleep(time.Duration(s.PreWaitS) * time.Second)` with log
      4. If `s.Name == "crash-recovery"`: call `r.runCrashRecovery(ctx, s)` instead of normal loadgen
      5. Otherwise: build and run loadgen cmd, wait for completion
      6. `r.writeMarker(s.Name, "end")`
      7. `time.Sleep(5 * time.Second)` — 5s drain window for in-flight CDC events before next scenario

    `(r *Runner) runCrashRecovery(ctx context.Context, s ScenarioDef) error`:
    - Start loadgen in background: `cmd.Start()` not `cmd.Run()`
    - Wait 30s for steady-state baseline
    - For each tool container in `[]string{"kaptanto", "debezium", "sequin", "peerdb"}`:
      - Record `killTime := time.Now()`
      - `exec.Command("docker", "kill", "--signal", "SIGKILL", container).Run()`
      - `exec.Command("docker", "start", container).Run()`
      - Poll `GET /scenario/last-event?tool=<container>` on collector URL every 500ms until timestamp advances past killTime or 120s timeout
      - Write `{"scenario_event":"recovery","tool":"X","recovery_seconds":N,"ts":"..."}` to metrics.jsonl
    - Wait for loadgen to finish

    Helper `setScenarioTag(ctx context.Context, name string) error`:
    - POST to `r.CollectorURL + "/scenario?name=" + name`
    - Return error on non-200

    Helper `writeMarker(scenario, event string)`:
    - Append single JSON line to `filepath.Join(r.OutputDir, "metrics.jsonl")`
    - JSON: `{"scenario_event": event, "scenario": scenario, "ts": time.Now().UTC()}`
    - Use `os.O_APPEND` — do not truncate existing file

    Note: The collector binary must also expose `GET /scenario/last-event?tool=X` returning the last receive_ts for that tool. This is needed by runCrashRecovery to detect delivery resumption. Add a note in the action that the collector binary (12-01) needs to expose this endpoint — the executor should add it to `bench/cmd/collector/main.go` if not already present.

    Write `runner_test.go` first. Tests use only pure functions and httptest servers — no Docker or loadgen binary needed.
