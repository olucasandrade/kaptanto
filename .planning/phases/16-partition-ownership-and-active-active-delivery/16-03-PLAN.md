**CRITICAL — shutdown ordering (Pitfall 2 in RESEARCH.md):**
The correct order is:
1. `g.Wait()` returns — this drains the Router (all runPartition goroutines exit) AND flushes cursors (PostgresCursorStore.Run exits, performing final flush)
2. After `g.Wait()` returns, call `pm.store.ReleaseAll(context.Background())`
3. After ReleaseAll, markOffline executes (NodeHeartbeater.markOffline is called in heartbeater.Run on ctx.Done(), which happens inside errgroup, so it already runs before g.Wait returns — this is correct)

The cleanest implementation: open PartitionStore and PartitionManager BEFORE the errgroup, use `defer pm.store.ReleaseAll(context.Background())` AFTER `defer heartbeater.Close(context.Background())` (defers run in LIFO: ReleaseAll runs FIRST, then heartbeater.Close).

Wait — `defer` runs after the function returns, but `g.Wait()` is called before the defers. So using `defer pm.store.ReleaseAll(...)` placed AFTER `defer heartbeater.Close(...)` means ReleaseAll runs before heartbeater.Close (LIFO). But we need ReleaseAll to happen AFTER g.Wait drains the Router.

Correct approach:
- Do NOT use defer for ReleaseAll.
- After `if err := g.Wait(); ...`, call `pm.store.ReleaseAll(context.Background())` explicitly:

