**Change 2 — Add `case "nats":` before the `default:` case:**

```go
case "nats":
    natsCfg := cfg.Sinks.NATS
    if natsCfg == nil {
        return fmt.Errorf("--output nats requires a sinks.nats block in config (url, subject-template)")
    }
    natsSink, err := natssink.NewNATSSinkConsumer("nats", natsCfg)
    if err != nil {
        return fmt.Errorf("nats sink: init: %w", err)
    }
    defer natsSink.Close()
    natsSink.SetMetrics(metrics)
    rtr.Register(natsSink)
    healthProbes = append(healthProbes, observability.HealthProbe{
        Name:  "nats",
        Check: natsSink.Ping,
    })
    // Re-build healthHandler after adding the nats probe.
    // NOTE: healthHandler was already constructed above; we need to append the probe
    // BEFORE healthHandler is built, OR reconstruct it here.
    // To keep changes minimal: move the healthProbes append BEFORE healthHandler is built.
    // SEE NOTE BELOW on placement.
    obsMux := http.NewServeMux()
    obsMux.Handle("/metrics", metrics.Handler())
    obsMux.Handle("/healthz", observability.NewHealthHandler(healthProbes))
    obsSrv := &http.Server{Addr: fmt.Sprintf(":%d", cfg.Port), Handler: obsMux}
    outputServer = func(ctx context.Context) error {
        go func() {
            <-ctx.Done()
            shutdownCtx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
            defer cancel()
            _ = obsSrv.Shutdown(shutdownCtx)
        }()
        if err := obsSrv.ListenAndServe(); err != nil && err != http.ErrServerClosed {
            return fmt.Errorf("nats obs server: %w", err)
        }
        return nil
    }
```

PLACEMENT NOTE: The `healthHandler` variable is created at line ~492 via `observability.NewHealthHandler(healthProbes)`. The NATS case must append the nats probe to `healthProbes` BEFORE `healthHandler` is constructed — OR the NATS case must reconstruct the health handler with the updated probes (as shown above with `observability.NewHealthHandler(healthProbes)` in the case block itself). Use the reconstruct approach so the existing code structure is not disturbed. The `healthProbes` slice at the time of the nats case will include all probes from the initial setup (eventlog, etc.) PLUS the nats probe just appended.
